How to Test the Trusted Types Monitor Extension

This guide helps you verify that Trusted Types Monitor is capturing trusted-types CSP violations and that the DevTools panel features (live stream, clustering, fix guide, policy generation, export, optional AI) work end-to-end.

Important: Trusted Types violations only happen on pages that enforce/report Trusted Types via CSP, e.g. Content-Security-Policy: require-trusted-types-for 'script' (or a matching Report-Only header). The included Test Page sets a CSP that requires Trusted Types.

Method 1: Using the Test Page

Step 1: Open the Test Page

Open the Test Page included with this repo.

If you open it as a local file: in Chrome go to chrome://extensions → open this extension’s details → enable Allow access to file URLs, otherwise the content script may not run on file:// pages.

Step 2: Open DevTools → Trusted Types Monitor panel

Open DevTools on the test page and select the Trusted Types Monitor panel (it may be under the » overflow menu). Ensure the Monitoring toggle is enabled.

Step 3: Generate violations

On the test page, click Run all unsafe tests (or click individual buttons). These trigger common unsafe sinks like innerHTML, insertAdjacentHTML, eval, and dynamic script injection.

Expected: the DevTools panel should start listing violations for the current tab. If CSP is enforcing, some buttons may throw TypeError; that’s OK—violations should still be observable/captured.

Step 4: Sanity check “safe” policy tests

Click Run all policy tests. These use a demo Trusted Types policy (named tt-monitor-demo) and should generally not add new Trusted Types violations (they may still fail if policy creation is blocked by CSP).

Method 2: Trigger a violation on any TT-enforced page

Step 1: Open a Web Page

Navigate to a page that enforces or reports Trusted Types via CSP (for example, an internal test environment or a site that already uses require-trusted-types-for 'script').

Step 2: Open the Developer Console

Right-click on the page, select "Inspect" or "Inspect Element", and navigate to the "Console" tab.

Step 3: Generate a Test Violation

Run one of the following commands (choose a sink that exists on your page):

// HTML sink (TrustedHTML)
document.body.insertAdjacentHTML('beforeend', '<img src=x onerror=console.log("tt-test")>');

// Script sink (TrustedScript)
eval('console.log("tt-test: eval")');
Expected: under enforcing CSP, these may throw (often TypeError). The goal is to trigger a Trusted Types CSP violation signal that the monitor captures for the tab.

Step 4: Check the DevTools panel

Open DevTools → Trusted Types Monitor. The Live table should show new rows for the current tab. Use the search box to filter by URL, directive, sample, or source file.

Method 3: Validate panel features (clusters, fixes, policies, export, AI)

Step 1: Clusters

After generating a handful of violations, open the Clusters tab. You should see grouped issues by type/root cause. Click Get Fix on a cluster to jump into the fix guide.

Step 2: Fix guide

In the Live table, click Fix on a row. The fix guide should show explanation, steps, and a copyable code example.

Step 3: Policy generator

Open the Policy tab and click Generate. Try both modes (the default policy generator and the “Perfect Types”/Sanitizer setHTML()-oriented mode, if present) and verify the output and warnings update.

Step 4 (optional): AI assistant

Open the AI tab, save an API key (and provider if applicable), then ask about a specific violation using Ask AI from the violation sidebar or fix guide.

Tip: You can use Export to download the captured violations for the current tab as JSON.

Troubleshooting

If you're not seeing any violations:

  • Make sure Monitoring is enabled in the DevTools panel
  • Make sure the page actually enforces/reports Trusted Types via CSP (the included test-page.html does)
  • If using file:// for the test page, enable Allow access to file URLs for the extension
  • Reload the page and keep DevTools open while generating violations
  • Check the Console for thrown TypeError or CSP-related messages (expected under enforcement)

If you continue to have issues, try restarting your browser or reinstalling the extension.
